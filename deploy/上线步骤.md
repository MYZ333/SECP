# 最快上线指南（照着做即可）

本目录是一键部署包。目标：**当天让 Web 管理端 + 后端接口在公网跑起来**。
小程序上线需要「备案 + HTTPS」，那部分见文末，先并行去备案。

---

## 一、买服务器（约 10 分钟）

去 **阿里云** 或 **腾讯云**，买一台「轻量应用服务器」：

- 配置：**2 核 4G** 起步（内存别选 2G，四个容器会不够）
- 系统镜像：选 **Docker** 应用镜像（腾讯云叫「Docker CE」，阿里云可选带宝塔的镜像）——这样连 Docker 都不用自己装
- 地域：国内用户选国内（如上海/北京）
- 价格：新用户活动价常年 ¥60–100/月

买完你会得到：一个 **公网 IP**、root 密码。

> 如果镜像里没自带 Docker，SSH 登录后跑一句就装好：
> `curl -fsSL https://get.docker.com | bash`

---

## 二、开放端口（2 分钟）

在云服务器控制台的「防火墙 / 安全组」里，放行这几个端口：

| 端口 | 用途 |
|------|------|
| 22   | SSH 登录（默认已开）|
| 80   | Web 网站（HTTP）|
| 443  | HTTPS（有域名后用）|

数据库 3306、Redis 6379 **不要对外开放**，容器内部互通即可，更安全。

---

## 三、上传项目 + 一键启动（约 10 分钟）

**1. 把整个项目传到服务器。** 最简单的方式：用 FinalShell / WinSCP / MobaXterm 这类工具（图形界面拖拽上传），把本地整个项目文件夹传到服务器的 `/root/hda` 目录。

**2. SSH 登录服务器，进入 deploy 目录，启动：**

```bash
cd /root/hda/deploy
docker compose up -d --build
```

首次会下载镜像 + 编译前后端，约 5–15 分钟。看到四个容器都 `Started` 就成功了。

**3. 检查是否都起来了：**

```bash
docker compose ps        # 四个容器状态应为 running / healthy
docker compose logs -f backend   # 看后端日志，出现 Started HdaApplication 即成功（Ctrl+C 退出）
```

**4. 浏览器访问：**

```
http://你的公网IP
```

用 `admin / 123456` 或 `user001 / 123456` 登录。**能登进去就说明后端+数据库+前端全通了。**

> 上线后请立刻登录管理端，把 admin 和 user001 的默认密码改掉。

---

## 四、常用运维命令

```bash
docker compose ps                 # 查看状态
docker compose logs -f backend    # 看后端日志
docker compose restart backend    # 只重启后端
docker compose down               # 停掉全部（数据保留在 volume 里）
docker compose up -d --build      # 改了代码后重新构建启动
```

## 五、小程序上线（需要备案，7–20 天，越早办越好）

小程序必须走 HTTPS + 备案域名，流程如下：

1. **买域名**（阿里云/腾讯云，约 ¥30/年），并在**买域名的同一家云**提交 **ICP 备案**。备案要 7–20 个工作日，**第一天就去提交**，这是最慢的一环。
2. 备案通过后，把域名解析到你的服务器公网 IP（加一条 A 记录）。
3. **配 HTTPS**：服务器装个「宝塔面板」，用它的 SSL 功能一键申请免费证书并开启 443（最省事）；或用 Certbot 命令行。
4. 改小程序接口地址：把 `miniprogram/src/api/request.js` 里的
   `const BASE_URL = 'http://localhost:8080/api'`
   改成 `const BASE_URL = 'https://你的域名/api'`，重新用 HBuilderX 打包。
5. **注册微信小程序账号**（mp.weixin.qq.com），在「开发管理 → 开发设置 → 服务器域名」里，把 `https://你的域名` 加进 **request 合法域名**白名单。
6. 用微信开发者工具导入 `miniprogram/dist/build/mp-weixin`，点「上传」，回到微信公众平台「提交审核」。审核 1–7 天，通过后点「发布」即正式上线。

---

## 六、上线前建议改的两处（非必须，但更专业）

1. **关掉调试日志**：`backend/src/main/resources/application.yml` 里
   `logging.level.com.medcare.hda: debug` 改成 `info`，
   MyBatis 的 `log-impl` 那行删掉或改为 `org.apache.ibatis.logging.nologging.NoLoggingImpl`。
2. **关掉在线接口文档**（防止接口暴露）：`knife4j.enable` 改成 `false`，
   或在 nginx.conf 里删掉 `/doc.html`、`/v3/`、`/webjars/` 三段 location。

密钥（JWT/AES/数据库密码）已在 `.env` 里帮你随机生成好，无需再改。

---

## 遇到问题

- **80 端口访问不了**：99% 是云服务器「安全组/防火墙」没放行 80，回第二步检查。
- **后端一直重启**：`docker compose logs backend` 看报错，多半是 MySQL 还没就绪（healthcheck 会自动等，稍等再看）或 `.env` 密码有特殊字符。
- **中文乱码 / Data too long**：本包已在 compose 里设置 utf8mb4，正常不会出现；若出现说明用了旧的数据卷，`docker compose down -v` 清掉数据卷重来（会清空数据）。
