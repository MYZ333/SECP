    # 后端代码讲解（按文件夹）

> 智慧医养 · 个人健康档案系统 · 后端 `com.medcare.hda`
> 技术栈：JDK 21 / Spring Boot 3.5.8 / Spring Security + JWT / MyBatis-Plus / MySQL 8 / Redis / Redisson / Spring AI(骨架) / Knife4j
> 本文按包（文件夹）逐个讲"是干什么的" + 主要代码分析。

---

## 0. 先看整体：一次请求怎么在后端流转

```
浏览器/小程序
   │  HTTP 请求 (带 Authorization: Bearer <token>)
   ▼
① TraceIdFilter          注入 traceId（日志全链路可追）
② GlobalRateLimitFilter  全局 QPS 兜底限流（超限直接 429）
③ Spring Security 过滤链
   └ JwtAuthenticationFilter  解析 token → 校验黑名单/登录态 → 注入登录用户
   ▼
④ Controller（@RestController）  接收参数、校验、组织返回
   ├ AOP 切面：@RateLimit 限流 / @Idempotent 幂等 / @DistributedLock 分布式锁
   ▼
⑤ Service（业务逻辑，@Transactional 事务）
   ▼
⑥ Mapper（MyBatis-Plus）  ←→ MySQL
   Service 也会读写 Redis（登录态/限流/缓存/锁/任务状态）
   ▼
⑦ 统一返回 Result<T>；异常由 GlobalExceptionHandler 兜底转成 Result
```

分层就是经典的 **Controller（接口层）→ Service（业务层）→ Mapper（数据层）**，加上 `security`（认证鉴权）、`aspect/annotation`（切面增强）、`config`（各种装配）、`common`（通用件）几组横切能力。

## 1. `com.medcare.hda`（根包 / 启动类）

**作用**：应用入口。

**`HdaApplication.java`**：
```java
@SpringBootApplication
@MapperScan("com.medcare.hda.mapper")   // 扫描所有 Mapper 接口，自动生成实现
public class HdaApplication {
    public static void main(String[] args) {
        SpringApplication.run(HdaApplication.class, args);
    }
}
```
`@SpringBootApplication` 开启自动装配；`@MapperScan` 让 MyBatis-Plus 扫描 `mapper` 包下的接口。跑 `main` 就启动了整个服务（内嵌 Tomcat，默认 8080）。

## 2. `config`（配置装配）

**作用**：把各种"基础设施"以 Bean 的形式装配进 Spring 容器。一个配置类干一件事。

| 文件 | 作用 |
|------|------|
| `MyBatisPlusConfig` | 注册**分页插件** `PaginationInnerInterceptor`，让 `page()` 查询能自动分页 |
| `MyMetaObjectHandler` | 插入/更新时**自动填充**公共字段：`create_time`、`update_time`、`deleted` |
| `RedisConfig` | 配 `RedisTemplate` 的序列化（key 用 String，value 用 JSON），存对象不乱码 |
| `RedissonConfig` | 装配 `RedissonClient`，用于**分布式锁**（手动装配，避免版本冲突） |
| `CacheConfig` | 开启 `@EnableCaching`，配 `RedisCacheManager`（医生详情缓存、TTL、防雪崩） |
| `AsyncConfig` | 开启 `@EnableAsync`，配**线程池**（有界队列 + 拒绝策略 + traceId 透传） |
| `Knife4jConfig` | 接口文档（Swagger/Knife4j），访问 `/doc.html` 在线调试 |
| `WebMvcConfig` | 把本地上传目录映射成 `/uploads/**`，让上传的头像能通过 URL 访问 |
| `DataEncryptMigrationRunner` | 敏感字段**存量加密迁移**（仅 `hda.crypto.migrate=true` 时跑一次） |

**主要代码分析 —— `MyMetaObjectHandler`**（省去手写时间戳）：
```java
@Override
public void insertFill(MetaObject metaObject) {
    this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
    this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    this.strictInsertFill(metaObject, "deleted", Integer.class, 0);   // 逻辑删除标记默认 0
}
```
配合实体 `BaseEntity` 上的 `@TableField(fill = FieldFill.INSERT)`，任何 `save()` 都会自动带上创建时间，不用每次手写。

**`AsyncConfig` 的关键点**——线程池 + 链路透传：
```java
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy()); // 队列满了让调用线程自己跑，防丢任务
executor.setTaskDecorator(runnable -> {          // 把主线程的 traceId 复制给异步线程
    Map<String,String> ctx = MDC.getCopyOfContextMap();
    return () -> { if (ctx != null) MDC.setContextMap(ctx); try { runnable.run(); } finally { MDC.clear(); } };
});
```

## 3. `security`（认证 / 授权 / 安全）

**作用**：谁能进、进来是谁、token 怎么发怎么校验——整套安全机制都在这里。

| 文件 | 作用 |
|------|------|
| `SecurityConfig` | Spring Security 主配置：白名单、无状态会话、把 JWT 过滤器插进链、密码用 BCrypt |
| `JwtUtil` | 生成/解析 JWT（access + refresh 双令牌，带 `jti` 便于登出黑名单） |
| `JwtProperties` | 读 `application.yml` 里 `hda.jwt.*` 配置（密钥、过期时间等） |
| `JwtAuthenticationFilter` | 每个请求解析 token → 查黑名单 → 比对 Redis 登录态 → 注入当前用户 |
| `RsaCryptoService` | 密码 RSA 传输加解密（前端加密、后端解密） |
| `LoginUser` | Spring Security 需要的 `UserDetails` 包装，承载当前登录用户 |
| `SecurityUtil` | 静态工具：`getUserId()` 随处取当前登录用户 ID |
| `JsonAuthEntryPoint` / `JsonAccessDeniedHandler` | 未登录(401)/无权限(403) 时返回 JSON（而非默认跳转页） |
| `TraceIdFilter` | 链路追踪：每请求生成 traceId 放进日志 MDC |
| `GlobalRateLimitFilter` | 全局 QPS 限流兜底 |
| `UserDetailsServiceImpl` | Security 加载用户的接口实现 |

**主要代码分析 —— `JwtAuthenticationFilter`（认证核心）**：
```java
Claims claims = jwtUtil.parseToken(token);
if (claims != null && 尚未认证) {
    String jti = claims.getId();
    // 1) 黑名单：登出的 token 立即失效
    if (redisTemplate.hasKey("hda:token:blacklist:" + jti)) { 返回 TOKEN_INVALID; return; }
    Long userId = claims.get("userId", Long.class);
    // 2) 单点登录：token 与 Redis 当前登录态不一致 → 被挤下线
    Object saved = redisTemplate.opsForValue().get("hda:login:token:" + userId);
    if (!Objects.equals(token, saved)) { 返回 LOGIN_ELSEWHERE; return; }
    // 3) 通过：把用户放进 SecurityContext，后续 @PreAuthorize / SecurityUtil 才拿得到
    SecurityContextHolder.getContext().setAuthentication(authentication);
}
```
这三步就是我们做的"登录加固"的落点：**黑名单**（登出即失效）、**单点登录**（新登录挤下旧的）、**注入身份**。

**`SecurityConfig` 的要点**——无状态 + 白名单 + 插入过滤器：
```java
.sessionManagement(sm -> sm.sessionCreationPolicy(STATELESS))  // 不用 Session，全靠 JWT
.authorizeHttpRequests(auth -> auth
    .requestMatchers(WHITE_LIST).permitAll()                   // 登录/注册/文档等放行
    .requestMatchers("/api/admin/**").hasRole("ADMIN")         // 管理端要 ADMIN 角色
    .anyRequest().authenticated())                             // 其余都要登录
.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
```

## 4. `common`（通用件）

**作用**：全局复用的基础类：统一返回、分页、状态码、体征规则，以及两个子包 `crypto`（加解密/脱敏）、`ratelimit`（限流器）。

| 文件 | 作用 |
|------|------|
| `Result<T>` | **统一响应体** `{code, message, data}`，所有接口都返回它 |
| `ResultCode` | 状态码枚举（200 成功、429 限流、1101 挤下线…），集中管理 |
| `PageQuery` / `PageResult` | 分页请求/结果的通用封装 |
| `MetricRules` | 体征阈值规则：录入血压/血糖时自动判断是否异常 |

**子包 `common/crypto`（数据安全）**：
| 文件 | 作用 |
|------|------|
| `FieldCrypto` | AES-256-GCM 字段加解密，密文带 `ENC:` 前缀，兼容遗留明文 |
| `EncryptTypeHandler` | MyBatis TypeHandler：写库自动加密、读库自动解密 |
| `Desensitize`(注解在 annotation 包) + `DesensitizeSerializer` + `DesensitizeType` | 出参脱敏（手机号 `138****8000`） |

**子包 `common/ratelimit`（限流）**：
| 文件 | 作用 |
|------|------|
| `RedisRateLimiter` | 加载 Lua 脚本，执行滑动窗口限流 |
| `LimitDimension` | 限流维度枚举：IP / USER / GLOBAL |

**主要代码分析 —— `Result<T>`（贯穿所有接口）**：
```java
public static <T> Result<T> success(String message, T data) { return build(200, message, data); }
public static <T> Result<T> fail(ResultCode rc) { return build(rc.getCode(), rc.getMessage(), null); }
```
好处：前端永远收到同一种结构，`code===200` 判成功，否则弹 `message`（前端拦截器就是这么写的）。

**`FieldCrypto`（加密落库）核心**：
```java
public static String encrypt(String plain) {
    if (plain 为空 || plain.startsWith("ENC:")) return plain;   // 已加密不重复加密（迁移幂等）
    byte[] iv = 随机12字节;                                     // GCM 每次随机 IV
    ... AES/GCM/NoPadding 加密 ...
    return "ENC:" + Base64(iv + 密文);
}
```

## 5. `annotation`（自定义注解）

**作用**：把"限流、幂等、锁、脱敏"做成一行注解，贴在方法/字段上即可，业务代码干净。真正的逻辑在 `aspect` 切面里。

| 注解 | 用法 | 效果 |
|------|------|------|
| `@RateLimit(key,window,limit,dimension)` | 贴在 Controller 方法 | 该接口按维度限流 |
| `@Idempotent` | 贴在提交类方法 | 校验幂等 token，防重复提交 |
| `@DistributedLock(key)` | 贴在 Service 方法 | Redisson 分布式锁，防并发 |
| `@Desensitize(type)` | 贴在实体字段 | JSON 输出时打码 |

**举例**：登录接口就一行注解实现"每 IP 每分钟最多 5 次"：
```java
@RateLimit(key = "login", window = 60, limit = 5, dimension = LimitDimension.IP)
@PostMapping("/login")
public Result<LoginVO> login(...) { ... }
```

## 6. `aspect`（AOP 切面）

**作用**：拦截贴了上面注解的方法，在方法执行前后插入通用逻辑。这是"注解生效"的地方。

| 文件 | 拦截谁 | 干什么 |
|------|--------|--------|
| `RateLimitAspect` | `@RateLimit` | 按维度拼 Redis key → 跑 Lua 滑动窗口 → 超限抛 429 |
| `IdempotentAspect` | `@Idempotent` | 原子消费请求头里的幂等 token，重复则拒绝 |
| `DistributedLockAspect` | `@DistributedLock` | 解析 SpEL 的锁 key → Redisson 加锁执行 → 释放 |

**主要代码分析 —— `RateLimitAspect`**：
```java
@Around("@annotation(rateLimit)")
public Object around(ProceedingJoinPoint pjp, RateLimit rateLimit) throws Throwable {
    String dim = resolveDimensionValue(rateLimit.dimension()); // IP=客户端IP, USER=用户ID
    String key = "hda:rl:" + rateLimit.key() + ":" + dim;
    if (!rateLimiter.tryAcquire(key, window毫秒, limit)) {      // Lua 原子判断
        throw new BusinessException(RATE_LIMIT_EXCEEDED);       // 超限 → 429
    }
    return pjp.proceed();                                       // 放行
}
```

**`DistributedLockAspect` 关键**——保证"加锁→事务→释放"顺序（`@Order(10)` 排在事务外层）：
```java
RLock lock = redissonClient.getLock("hda:lock:" + spel解析后的key);
if (!lock.tryLock(waitSeconds, leaseSeconds, SECONDS)) throw new BusinessException(LOCK_ACQUIRE_FAIL);
try { return pjp.proceed(); } finally { if (持锁) lock.unlock(); }
```

## 7. `controller`（接口层）

**作用**：定义 REST 接口，接收/校验参数，调用 Service，返回 `Result`。**不写业务逻辑**，只做编排。按业务分了 16 个。

| Controller | 负责 |
|------|------|
| `AuthController` | 登录/注册/短信/忘记密码/**刷新令牌/登出/RSA 公钥** |
| `AccountController` | 个人资料、改密、注销账号 |
| `HealthProfileController` | 健康档案基本信息 |
| `HealthMetricController` | 血压/血糖等体征记录 |
| `MedicalRecordController` | 就诊/用药记录 |
| `HealthReportController` | 健康报告 |
| `AlertController` | 健康预警 |
| `DoctorController` | 医生专家库（列表/科室/详情） |
| `ConsultController` | AI 健康咨询 |
| `PointController` | 积分：余额/签到/**任务领取**/明细/商城/兑换 |
| `IdempotentController` | 领取幂等令牌 |
| `FileController` | 文件（头像）上传 |
| `AdminUser/Doctor/Product/ExchangeController` | 管理端维护 |

**主要代码分析 —— `PointController` 的任务领取（我们新加的三态领取）**：
```java
@Idempotent                                  // 幂等：兑换等提交防重复
@PostMapping("/exchange")
public Result<PointExchange> exchange(@Valid @RequestBody ExchangeDTO dto) {
    return Result.success("兑换成功", pointService.exchange(SecurityUtil.getUserId(), dto));
}

@PostMapping("/claim/{type}")                // 待领取 → 已完成
public Result<ClaimVO> claim(@PathVariable String type) {
    return Result.success("领取成功", pointService.claimTask(SecurityUtil.getUserId(), type));
}
```
注意：`SecurityUtil.getUserId()` 从 SecurityContext 取当前用户，接口不用前端传 userId，安全。

## 8. `dto`（数据传输对象）

**作用**：接口的**入参**（DTO）和**出参**（VO）。和数据库实体分开，避免把内部字段暴露给前端，也方便加校验。

- 入参 DTO：`LoginDTO`、`RegisterDTO`、`ChatDTO`、`ExchangeDTO`、`TokenRefreshDTO`…（带 `@NotBlank`/`@Pattern` 等校验）
- 出参 VO：`LoginVO`（token+refreshToken+用户信息）、`PointTaskVO`（任务及**status**）、`ClaimVO`（领取结果）…

**主要代码分析 —— `LoginDTO` 的参数校验**：
```java
@NotBlank(message = "用户名不能为空")
private String username;
@NotBlank(message = "密码不能为空")
private String password;
```
配合 Controller 的 `@Valid`，参数不合法会被 `GlobalExceptionHandler` 统一转成友好提示，不会进业务代码。

## 9. `entity`（数据库实体）

**作用**：和数据库表一一对应的类，MyBatis-Plus 用它做 CRUD。

| 实体 | 表 |
|------|----|
| `User` | `sys_user` 用户 |
| `HealthProfile` | 健康档案（过敏史等**加密字段**） |
| `HealthMetric` | 体征数据 |
| `MedicalRecord` | 就诊记录（诊断/处方**加密字段**） |
| `HealthReport` / `HealthAlert` | 报告 / 预警 |
| `Doctor` | 医生 |
| `ConsultRecord` | AI 咨询消息 |
| `PointRecord` / `PointProduct` / `PointExchange` | 积分流水 / 商品 / 兑换 |
| `BaseEntity` | 公共父类：create_time/update_time/deleted |

**主要代码分析 —— `MedicalRecord` 的加密与实体映射**：
```java
@TableName(value = "medical_record", autoResultMap = true)   // autoResultMap 让字段级 typeHandler 生效
public class MedicalRecord extends BaseEntity {
    @TableId(type = IdType.AUTO) private Long id;
    @TableField(typeHandler = EncryptTypeHandler.class)       // 写库加密、读库解密
    private String diagnosis;
    @TableField(typeHandler = EncryptTypeHandler.class)
    private String prescription;
}
```
`BaseEntity` 里 `@TableLogic private Integer deleted;` 实现**逻辑删除**——删除只是把 `deleted` 置 1，查询自动过滤，不真删数据。

## 10. `mapper`（数据访问层）

**作用**：MyBatis-Plus 的 Mapper 接口。继承 `BaseMapper<T>` 就白得增删改查、分页，一般**不用写 SQL**。

public interface UserMapper extends BaseMapper<User> { }   // 就这一行，CRUD 全有了

## 11. `service` + `service/impl`（业务层）

**作用**：真正的业务逻辑。`service` 是接口，`service/impl` 是实现。事务 `@Transactional` 加在这一层。

几个重点实现：

**`AuthServiceImpl`（认证业务）**——发双令牌 + 单点登录：
```java
private LoginVO issueToken(User user) {
    String access  = jwtUtil.generateToken(...);          // 30 分钟
    String refresh = jwtUtil.generateRefreshToken(...);   // 7 天
    // 单点登录：每人只存一个有效 token，新登录覆盖旧的 → 旧端被挤下线
    redisTemplate.opsForValue().set("hda:login:token:"+id, access, 过期, MILLISECONDS);
    redisTemplate.opsForValue().set("hda:login:refresh:"+id, refresh, 过期, MILLISECONDS);
    pointService.markTaskReady(id, "LOGIN");              // 登录任务 → 待领取
    return LoginVO...;
}
```

**`PointServiceImpl`（积分业务）**——分布式锁 + 三态领取：
```java
@DistributedLock(key = "'exchange:' + #userId")           // 防并发超卖
@Transactional
public PointExchange exchange(Long userId, ExchangeDTO dto) { 校验积分/库存 → 扣减 → 落记录 }

@Transactional
public ClaimVO claimTask(Long userId, String type) {
    if (!redis.hasKey(readyKey)) throw 任务未完成;          // 必须"待领取"
    Boolean first = redis.opsForValue().setIfAbsent(claimedKey, "1", ...); // 原子占位防重复领取
    if (!first) throw 已领取;
    PointRecord r = addPoints(userId, 分值, type, 描述);   // 发分
    return ClaimVO(分值, r.getBalance());
}
```

**`AiChatServiceImpl`（AI 骨架）**：注入 Spring AI 的 `ChatClient`，配了模型 key 就真调用，没配就返回占位文案（保证没接模型也能跑）：
```java
ChatClient.Builder builder = chatClientBuilder.getIfAvailable();
if (builder == null) return placeholder(message);        // 未配置 → 占位
return builder.build().prompt().system(SYSTEM_PROMPT).user(message).call().content();
```

**`AsyncTaskServiceImpl`**：`@Async` 把"标记任务待领取"丢到线程池异步做，不阻塞 AI 应答主链路。

## 12. `exception`（异常处理）

**作用**：把各种异常统一转成友好的 `Result`，不让堆栈/SQL 泄露给前端。

| 文件 | 作用 |
|------|------|
| `BusinessException` | 自定义业务异常，带 code + message，业务里 `throw new BusinessException(...)` |
| `GlobalExceptionHandler` | `@RestControllerAdvice` 全局兜底，分类处理各种异常 |

**主要代码分析 —— `GlobalExceptionHandler`**：
```java
@ExceptionHandler(BusinessException.class)              // 业务异常 → 原样返回 code/message
public Result<Void> handleBusiness(BusinessException e) { return Result.fail(e.getCode(), e.getMessage()); }

@ExceptionHandler(MethodArgumentNotValidException.class) // 参数校验失败 → 取第一条提示
public Result<Void> handleValid(BindException e) { ... }

@ExceptionHandler(Exception.class)                       // 兜底 → 只说"系统繁忙"，细节记日志
public Result<Void> handleException(Exception e) { log.error("系统异常", e); return Result.fail(...); }
```

## 14. `resources`（配置与脚本，非 Java）

| 文件 | 作用 |
|------|------|
| `application.yml` | 主配置：数据库、Redis、JWT、限流阈值、加密密钥、短信、AI |
| `logback-spring.xml` | 日志格式（带 `[traceId]`）、分文件滚动 |
| `schema.sql` | 建库建表 + 初始数据（20 位医生等） |
| `lua/sliding_window_rate_limit.lua` | 滑动窗口限流脚本（原子判断+计数+过期） |
| `lua/idempotent_check.lua` | 幂等校验脚本（原子"存在即消费"） |
| `mapper/*.xml` | 需要手写 SQL 时放这里 |
| `data_security_upgrade.sql` / `dedup_doctors.sql` / `reset_doctors.sql` / `add_doctors.sql` | 我们加的运维脚本（字段扩容/去重/重置医生） |

**主要代码分析 —— 限流 Lua（为什么用 Lua）**：
```lua
redis.call('ZREMRANGEBYSCORE', key, 0, now - window)   -- 清理窗口外旧记录
local count = redis.call('ZCARD', key)
if count < limit then
    redis.call('ZADD', key, now, member)                -- 记录本次
    redis.call('PEXPIRE', key, window)
    return 1                                             -- 放行
else return 0 end                                       -- 拒绝
```
用 Lua 是因为"判断+计数+过期"必须**原子**执行，否则高并发下会漏限。

## 15. 一句话总结每个文件夹

| 文件夹 | 一句话 |
|--------|--------|
| （根） | 启动入口 |
| `config` | 把基础设施装进容器（分页/缓存/线程池/锁/文档） |
| `security` | 谁能进、进来是谁（JWT + 认证鉴权 + 安全过滤） |
| `common` | 统一返回/状态码/分页 + 加密脱敏 + 限流器 |
| `annotation` | 把限流/幂等/锁/脱敏做成一行注解 |
| `aspect` | 让上面注解真正生效的切面逻辑 |
| `controller` | 接口层，收参数、调 Service、返 Result |
| `dto` | 接口的入参/出参对象（与实体解耦） |
| `entity` | 数据库表映射类 |
| `mapper` | 数据访问接口（MyBatis-Plus 自动 CRUD） |
| `service`(+impl) | 业务逻辑与事务 |
| `exception` | 异常统一转友好返回 |
| `util` | 通用小工具 |
| `resources` | 配置、SQL、Lua、日志 |

**关注顺序建议**：先看 `entity` → `mapper`（数据长啥样）→ `controller` → `service`（业务怎么跑）→ `security`（怎么保证安全）→ `aspect/annotation`（限流幂等锁）→ `config`（怎么装配起来）。
